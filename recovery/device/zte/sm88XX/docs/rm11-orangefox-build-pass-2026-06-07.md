The RM11 Pro / RedMagic 11 Pro OrangeFox recovery tree now builds successfully against a synced OrangeFox `fox_14.1` source tree.

## Build Fixes Applied

- Synced OrangeFox `fox_14.1` source to `<orangefox-tree>`.
- Copied the RM11 device tree into `<orangefox-tree>/device/nubia/NX809J`.
- Updated `AndroidProducts.mk` lunch choices to the Android 14 release-qualified form:
  - `orangefox_NX809J-ap2a-eng`
  - `orangefox_NX809J-ap2a-userdebug`
- Kept live Android 16/API 36 identity in docs and `system.prop`, but stopped forcing `BOARD_SHIPPING_API_LEVEL=36` / `PRODUCT_SHIPPING_API_LEVEL=36` in the Android 14 recovery build. The current OrangeFox/AP2A build base only supports SystemSDK versions through 34.
- Built with `mka -j8` to avoid the high parallelism warning on a 23 GB RAM WSL environment.
